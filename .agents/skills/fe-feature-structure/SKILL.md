# Frontend Feature Structure

## Overview

The frontend (`automotive.marketplace.client/src/`) is divided into distinct layers. Understanding where each file lives is critical — agents commonly place files in the wrong layer.

## Full `src/` Directory Map

```
src/
  app/
    pages/        ← Page components (UI assembled from feature components)
    routes/       ← TanStack Router route definitions (thin config, no UI logic)
    App.tsx
    main.tsx
  features/       ← Self-contained feature modules (most code lives here)
  components/
    ui/           ← shadcn/ui primitives (Button, Input, Dialog, etc.)
    common/       ← Custom shared primitives used across features
    forms/        ← Shared form inputs used across features
    layout/       ← App-wide layout (Header, etc.)
    providers/    ← React context providers (ThemeProvider, etc.)
  api/
    <resource>/   ← queryOptions() for calls shared by 2+ features
    queryKeys/    ← ALL TanStack Query key factories live here
  types/          ← TypeScript types shared across features
  constants/      ← App-wide constants (endpoints, validation limits, etc.)
  lib/
    axios/        ← Axios instances and interceptors
    redux/        ← Redux store root
    tanstack-query/ ← QueryClient config
    router.tsx    ← TanStack Router instance
    utils.ts      ← shadcn cn() utility
  hooks/          ← Shared hooks (typed Redux hooks, etc.)
  utils/          ← Shared pure utility functions
```

## CRITICAL: Routes vs Pages

This is where agents make mistakes. There are **two separate files** per page:

**`src/app/routes/listings.tsx`** — Route definition only. Thin config file:
- Calls `createFileRoute()`
- Defines `loader`, `validateSearch`, `loaderDeps`, etc.
- Imports and sets the page component via `component:`
- Has no UI markup

**`src/app/pages/Listings.tsx`** — Page component. The actual UI:
- A plain React component
- Composes feature components (imports from feature `index.ts`)
- Uses `Route.useSearch()`, `Route.useNavigate()` etc. from the route file

```ts
// src/app/routes/listings.tsx  ← ROUTE CONFIG
import Listings from "@/app/pages/Listings";
export const Route = createFileRoute("/listings")({
  component: Listings,
  loader: async ({ context: { queryClient } }) => { ... },
});

// src/app/pages/Listings.tsx  ← PAGE COMPONENT
import { Route } from "@/app/routes/listings";
import { ListingList, Filters } from "@/features/listingList";
const Listings = () => {
  const searchParams = Route.useSearch();
  return <ListingList listingSearchQuery={searchParams} />;
};
```

**Rule:** When adding a new page — create BOTH files. The page component goes in `src/app/pages/`. The route config goes in `src/app/routes/`. **Never put UI layout in the route file.**

**Never edit `src/routeTree.gen.ts`** — it is auto-generated by TanStack Router.

## Feature Module Layout

```
src/features/listingList/          # camelCase feature name
  api/
    getAllListingsOptions.ts        # queries → queryOptions() wrapper
    useDeleteListing.ts            # mutations → useMutation() hook
  components/
    ListingList.tsx                # React components (PascalCase files)
    ListingCard.tsx
  schemas/
    listingFilterSchema.ts         # Zod schemas for forms in this feature
  types/
    GetAllListingsQuery.ts         # TS types mirroring BE command/query/response shapes
    GetAllListingsResponse.ts
  state/                           # Only if feature needs Redux slice
    listingSlice.ts
  utils/                           # Only if feature has non-trivial pure functions
    formatListingPrice.ts
  index.ts                         # Public API — explicit named exports only
```

**Not every subfolder is required.** Only create what the feature needs.

## index.ts — Public API

Only export what other features or pages actually import:

```ts
// src/features/listingList/index.ts
export { getAllListingsOptions } from "./api/getAllListingsOptions";
export { default as ListingList } from "./components/ListingList";
export { default as Filters } from "./components/Filters";
export type { GetAllListingsQuery } from "./types/GetAllListingsQuery";
```

Never re-export everything with `export * from`. Keep the surface intentional.

## Where Does It Go? Decision Table

| What you're creating | Where it goes |
|---|---|
| New page's UI layout | `src/app/pages/MyPage.tsx` |
| New page's route config | `src/app/routes/my-page.tsx` |
| Feature-specific React component | `src/features/<name>/components/` |
| Feature-specific API call (query or mutation) | `src/features/<name>/api/` |
| API call shared by 2+ features | `src/api/<resource>/` |
| Feature-specific Zod schema | `src/features/<name>/schemas/` |
| Feature-specific TS types | `src/features/<name>/types/` |
| TS types shared across features | `src/types/` |
| Feature Redux slice | `src/features/<name>/state/` |
| Redux store wiring | `src/lib/redux/store.ts` |
| All TanStack Query key factories | `src/api/queryKeys/` |
| shadcn/ui primitive (Button, etc.) | `src/components/ui/` |
| Custom shared UI primitive | `src/components/common/` |
| Shared form input (used in 2+ features) | `src/components/forms/` |
| App-wide layout (header, nav) | `src/components/layout/` |
| App-wide constants | `src/constants/` |
| Shared pure utility function | `src/utils/` |
| Shared custom hook | `src/hooks/` |

## Types

Mirror backend shapes exactly:

```ts
// GetAllListingsResponse.ts
export type GetAllListingsResponse = {
  id: string;
  price: number;
  thumbnailUrl: string | null;
};
```

## Schemas

Zod schemas go in `schemas/`. Use `VALIDATION` constants and `validation` helpers:

```ts
import { VALIDATION } from "@/constants/validation";
import { validation } from "@/utils/validation";
import { z } from "zod";

export const MyFeatureSchema = z.object({
  price: z.coerce.number().min(VALIDATION.PRICE.MIN, {
    error: validation.minSize({ label: "Price", size: VALIDATION.PRICE.MIN, unit: "€" }),
  }),
});
```

## Common Mistakes

| Mistake | Fix |
|---|---|
| Putting page component inside `src/app/routes/` | Move UI to `src/app/pages/`, keep only route config in routes |
| Putting page component inside a feature | Pages live in `src/app/pages/`, not in features |
| Creating shared component inside a feature | If 2+ features need it, move to `src/components/` |
| Creating query keys inside a feature | All query keys go in `src/api/queryKeys/` |
| Editing `routeTree.gen.ts` | Never — it's auto-generated |
